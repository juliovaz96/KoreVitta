# KoreVitta — UX Writing System (Glossário)

## v1.0 — pt-BR oficial

Este documento define o vocabulário único da interface da KoreVitta para garantir consistência em todas as telas P0/P1.

## 1) Princípios de escrita
- Clareza primeiro: frases curtas, diretas e acionáveis.
- Linguagem humana: evitar jargão técnico quando houver alternativa simples.
- Consistência: o mesmo conceito deve ter sempre o mesmo nome.
- Ação explícita: botões com verbo de ação (`Salvar`, `Enviar`, `Continuar`).
- Tom da marca: profissional, acolhedor e objetivo.

## 2) Padrões obrigatórios
- Idioma padrão: pt-BR.
- Termos com hífen: `check-in`.
- Uso de maiúsculas em títulos: apenas a primeira palavra (sentence case), salvo siglas.
- Erros: explicar o problema e oferecer próximo passo (`Tentar novamente`).

## 3) Dicionário oficial (UI)
- `Dashboard CRM` (manter)
- `Pacientes` (manter)
- `Mensagens` (manter)
- `Check-ins` (manter)
- `Templates` (manter no produto)
- `Analytics` -> `Painel de análises`
- `Branding` -> `Identidade visual`
- `Subscription / Upgrade` -> `Assinatura e upgrade`
- `Threads` -> `Conversas`
- `Chat ativo` -> `Conversa ativa`
- `Home` (paciente) -> `Início`
- `Loading` -> `Carregamento`
- `Empty` -> `Sem dados`
- `Error` -> `Erro`

## 4) Padrões de CTA
- Primário: verbo + resultado (`Criar e enviar convite`, `Ativar protocolo`, `Enviar check-in`).
- Secundário: orientação de navegação (`Voltar para pacientes`, `Ir para Hoje`).
- Evitar genéricos como `OK`, `Submit`, `Apply`.

## 5) Templates de feedback
- Sucesso: `Ação concluída com sucesso.`
- Erro: `Não foi possível concluir a ação. Tente novamente.`
- Vazio: `Ainda não há dados para exibir aqui.`
- Permissão: `Você não tem acesso a esta área.`

## 6) Escopo desta versão
- Aplicação nas telas P0/P1 do frontend atual.
- Revisão contínua a cada sprint com atualizações versionadas.
